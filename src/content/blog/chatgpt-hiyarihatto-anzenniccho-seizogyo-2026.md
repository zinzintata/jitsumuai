## 3. ヒヤリハット報告書をChatGPTで作る方法

### 報告書の5要素

ヒヤリハット報告書は以下の5要素で構成されます。

- **発生日時・場所・作業内容:** いつ、どこで、何をしていたときに起きたか
- **事象の詳細:** 何が起きそうだったか（ケガには至らなかった状況）
- **直接原因:** その事象がなぜ起きたか（直接のきっかけ）
- **背後要因:** 管理・環境・教育上の問題として、なぜ防げなかったか
- **再発防止策:** 同じことが起きないようにするために何をするか

この5要素をChatGPTに渡すだけで、読み手に伝わる報告書の下書きが完成します。

### コピペOKプロンプト①：ヒヤリハット報告書（基本版）

```
製造業向けのヒヤリハット報告書の下書きを作成してください。

【発生日時・場所・作業内容】
・発生日時：[例：2026年4月22日 午前10時頃]
・発生場所：[例：第1倉庫 資材搬入エリア]
・作業内容：[例：フォークリフトで資材パレットを棚の最上段に積み上げる作業]

【事象の詳細（何が起きそうだったか）】
・[例：フォークリフトで荷物を持ち上げた際、バランスが崩れパレットが傾いた。荷物が落下寸前になったが、すぐに気づいて操作を止めた。落下には至らなかった。]

【直接原因（推定）】
・[例：パレットの中心に荷物を積まずに偏って積載していた]
・[例：積載状態の確認をせずに持ち上げ操作を開始した]

【背後要因（管理・環境上の問題）】
・[例：積載確認の手順が作業標準書に明記されていなかった]
・[例：高積みパレットの積載基準が口頭伝達のみで周知されていた]

【再発防止策（案）】
・[例：パレット積載基準を作業標準書に明記し、全作業者に周知する]
・[例：高積み作業前の確認チェックリストを作成・運用する]

上記を「1. 発生状況 / 2. 事象の詳細 / 3. 直接原因 / 4. 背後要因 / 5. 再発防止策」の
構成で、社内安全管理書類として適切な文体でまとめてください。
事実は断定せず「〜と推定される」「〜の可能性がある」という表現を使ってください。
```

### コピペOKプロンプト②：背後要因・再発防止策の深掘り

「原因は書けたが、再発防止策が薄い」と感じたときに使うプロンプトです。

```
以下のヒヤリハット報告書の「背後要因」と「再発防止策」をより深く掘り下げてください。

【現在記載している直接原因】
[例：作業者が積載確認を怠った]

【背後要因の深掘り依頼】
「作業者の不注意」という個人責任に帰結させず、
「管理の仕組みとして何が欠けていたか」という観点で背後要因を3点挙げてください。

【再発防止策の深掘り依頼】
「注意を徹底する」ではなく、
「仕組みとして再発を防ぐ」具体的な対策を3点提案してください。
（例：標準書改訂/チェックリスト導入/設備改善/教育計画など）
```

### 「原因が浅い」よくある失敗とAIで直す方法

ヒヤリハット報告書でよく見られる「原因分析が浅い」失敗例と、AIでの改善方法を示します。

**❌ 浅い原因記述の例:**
> 「作業者が確認を怠った」「不注意だった」

**✅ 深い原因記述の例:**
> 「積載確認の手順が作業標準書に明記されておらず、確認するかどうかが作業者の判断に委ねられていた。また、新人への引き継ぎ教育で当該リスクの説明が不十分だった」

「原因が浅い」報告書は、安全パトロールや監査で指摘を受けやすくなります。AIに「この原因記述は管理システムの視点から深掘りしてほしい」と依頼することで、次の指摘を防ぐレベルまで引き上げられます。

> 無料版でも報告書の下書きは作成できますが、
> 「なぜ管理の仕組みとして防げなかったか」という
> 背後要因の深掘りや、再発防止策の具体化では
> ChatGPT Plusの論理的な思考力が大きな差を生みます。

## 4. 安全日報をChatGPTで効率化する方法

安全日報は毎日または週1回作成する定例書類です。「毎回同じことしか書けない」「パトロールで見たことを文章にするのが苦手」という担当者が多い書類でもあります。AIを使えば、見回りで気になった点を箇条書きで入力するだけで、読み手に伝わる安全日報の下書きが完成します。

### 安全日報の構造

```
1. 作業内容・担当エリア（今日どこで何をしたか）
2. 安全確認事項（危険箇所・注意点）
3. ヒヤリハット・異常の有無（今日発生したか）
4. 安全対策の実施状況（KY・保護具・整理整頓等）
5. 翌日の作業予定・注意点
```

### コピペOKプロンプト③：安全日報

```
製造業の安全日報の下書きを作成してください。

【作業日・担当者・エリア】
・日付：[例：2026年4月22日]
・担当エリア：[例：第2製造ライン 組立工程・梱包工程]

【今日の作業内容】
・[例：午前：組立工程の定期清掃と設備点検]
・[例：午後：梱包資材の棚卸し作業]

【安全確認事項・気になった点】
・[例：組立工程の床に油漏れの跡があった。清掃済みだが発生源を要確認]
・[例：梱包エリアで通路に資材が一時的に置かれており、転倒リスクあり。作業後に片付け済み]

【ヒヤリハット・異常の有無】
・[例：本日のヒヤリハットなし]
または
・[例：〇〇工程で△△のヒヤリハットが1件発生。別途ヒヤリハット報告書を提出予定]

【翌日の作業予定・注意点】
・[例：外部業者による設備メンテナンス立会い（午前9時〜）]
・[例：業者作業中は該当エリアへの立ち入り制限を周知する]

上記を「1. 作業概要 / 2. 安全確認事項 / 3. ヒヤリハット・異常 / 4. 安全対策実施状況 / 5. 明日の予定と注意点」の構成でまとめてください。
上司・安全管理責任者への報告として適切な文体でお願いします。
```

## 5. KY（危険予知）活動記録をChatGPTで作る方法

KY活動は作業前に「今日の作業にどんな危険が潜んでいるか」をチームで確認する安全活動です。毎日の短時間ミーティングで実施しますが、その記録を毎回作成するのは手間がかかります。

### コピペOKプロンプト④：KY活動記録

```
製造業のKY（危険予知）活動記録の下書きを作成してください。

【本日の作業内容】
・[例：高所（4m）での照明器具の交換作業]
・[例：作業人数：2名（作業者1名・補助者1名）]

【想定される危険・リスク（思いついたもの）】
・[例：高所からの転落・落下]
・[例：使用する工具の落下による下への影響]
・[例：脚立の転倒]

上記を元に以下を作成してください:
1. 本日の作業に潜む危険ポイント（3〜4点）
2. 各危険ポイントに対する安全対策・行動目標
3. 本日の安全スローガン（一言で覚えやすいもの）

KYミーティングの記録用紙に記載できる形式でまとめてください。
```
