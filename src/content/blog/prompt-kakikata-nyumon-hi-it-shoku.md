# プロンプトの書き方入門【非IT職向け・5つの型とコピペ例文付き】

「ChatGPTに聞いてみたけど、なんか的外れな答えが返ってきた」——そんな経験はありませんか。実は問題はAIの性能ではなく、**指示の出し方（プロンプト）**にあることがほとんどです。この記事では、IT知識ゼロの製造業・事務職の方でも今日から使える「5つのプロンプトの型」と、職場でそのままコピペして使える例文を紹介します。

---

## 「うまく答えが返ってこない」3つの原因

まず、なぜAIへの質問がうまくいかないのかを確認しましょう。

### 原因①: 情報が少なすぎる

```
悪い例: 「報告書を書いて」
良い例: 「今週の生産進捗について、上司向けの週次報告書を300字で書いて」
```

「誰向けか」「何について」「どんな形式で」が抜けると、AIは汎用的すぎる回答を返します。

### 原因②: 目的が曖昧

```
悪い例: 「メールを直して」
良い例: 「以下のメールを、丁寧だが簡潔なビジネス文書として書き直して。文章は3行以内に」
```

「直す」だけでは何を改善したいかがAIに伝わりません。

### 原因③: 出力形式を指定していない

```
悪い例: 「4M会議の注意点を教えて」
良い例: 「4M会議で確認すべき注意点を、製造現場の担当者向けに箇条書き5項目でまとめて」
```

形式を指定するだけで、使える答えが返ってくる確率が大きく上がります。

---

## 今日から使える5つのプロンプトの型

プロンプトには「型（パターン）」があります。5つの型を覚えるだけで、あらゆる業務に応用できます。

---

### 型①:「役割型」——専門家に聞く

**使い方:** AIに役割・立場を与えることで、その分野に特化した回答を引き出す。

**基本フォーマット:**
```
あなたは[役割]です。[依頼内容]
```

**製造業・事務職の実例:**
```
あなたは製造業の品質管理の専門家です。
不良品が発生した際の初動対応チェックリストを10項目で作成してください。
```

```
あなたは経験豊富なビジネス文書の校正者です。
以下の報告書の文章を、わかりやすく簡潔に書き直してください。
[文章をここに貼る]
```

**ポイント:** 役割は具体的なほど効果的。「専門家」より「製造業の品質管理歴10年の専門家」の方が精度が上がります。

---

### 型②:「状況型」——背景を伝える

**使い方:** 「こういう状況だから、こうしてほしい」と文脈を与える。

**基本フォーマット:**
```
[状況・背景]。[依頼内容]
```

**製造業・事務職の実例:**
```
来週、取引先A社に工場見学に来てもらいます。
先方の担当者は製造業の知識はあるが、当社の工程は初見です。
見学当日に説明する工程紹介の資料の構成案を作ってください。
```

```
新入社員向けの業務マニュアルを作成中です。
受注管理システムへのデータ入力手順を、PC操作に不慣れな人向けに
ステップバイステップで説明してください。
```

**ポイント:** 「誰が読むか」「どんな目的で使うか」を伝えると、トーンや内容が自動的に最適化されます。

---

### 型③:「出力型」——形式を指定する

**使い方:** 「どんな形で出してほしいか」を最初に決める。

**基本フォーマット:**
```
[依頼内容]を、[形式]・[文字数/項目数]・[対象読者]向けで出力してください
```

**製造業・事務職の実例:**
```
安全パトロールの結果報告を、以下の形式で作成してください。
- 形式: 表形式（指摘箇所・危険度・対応期限の3列）
- 項目数: 5件まで
- 対象: 工場長への報告用
```

```
月次の売上集計結果を報告するメールを作成してください。
- 文字数: 200字以内
- トーン: 丁寧・簡潔
- 宛先: 部門長
- 形式: 件名＋本文の構成で
```

**ポイント:** 「箇条書き / 表形式 / 〇字以内 / 見出し付き」など、形式を具体的に指定するほど使いやすい出力になります。

---

### 型④:「制約型」——いらないものを削る

**使い方:** 「〜はしないで」「〜だけに絞って」と制限を加える。

**基本フォーマット:**
```
[依頼内容]。ただし、[制約条件]
```

**製造業・事務職の実例:**
```
ChatGPTを業務で活用するメリットを説明してください。
ただし、専門用語は使わず、製造業の現場作業者にもわかる言葉で書いてください。
```

```
以下の議事録を要約してください。
ただし、決定事項とアクションアイテムだけを抽出してください。
雑談・議論の経緯は含めないでください。
[議事録テキストを貼る]
```

**ポイント:** 「〜しないで」という否定形の指示は特に効果的。出力のノイズを大幅に減らせます。

---

### 型⑤:「例示型」——見本を見せる

**使い方:** 「こんな感じで」と具体例を見せて、同じパターンで出力させる。

**基本フォーマット:**
```
以下の例を参考に、[依頼内容]を作成してください。
例: [サンプル]
```

**製造業・事務職の実例:**
```
以下の形式で、今週の日報を作成してください。

【例】
■ 実施業務
・受注データの入力（件数: 15件）
・週次集計表の更新
■ 翌日の予定
・棚卸し準備（9:00〜12:00）
■ 連絡事項
・なし

今週の内容:
- 品質会議の議事録作成（2時間）
- 取引先への見積書送付（3件）
- システム不具合の報告書作成
```

```
以下の件名・文体を真似て、納品遅延のお詫びメールを書いてください。

【例のメール】
件名: 〇〇の件について（ご連絡）
本文: お世話になっております。〇〇株式会社の△△でございます。
この度は〜について、ご迷惑をおかけし誠に申し訳ございません。
[以下続く]
```

**ポイント:** 例が具体的なほど、出力のブレがなくなります。社内フォーマットをそのまま「例」として貼り付けるのが最も効果的です。

---

## 組み合わせると精度が10倍上がる

5つの型を**組み合わせる**のが最も強力です。実際の業務では、1つの型だけより複数を組み合わせた方がはるかに精度の高い回答が得られます。

**組み合わせプロンプト例（役割型 + 状況型 + 出力型 + 制約型）:**

```
あなたは製造業の安全管理の専門家です。（役割型）

当社は従業員50名の部品加工工場で、来月から新しい設備を導入します。
導入に際して、作業者向けの安全教育を実施する予定です。（状況型）

安全教育で伝えるべき重要ポイントを、以下の形式で出力してください。（出力型）
- 形式: 箇条書き
- 項目数: 7つ以内
- 対象: 設備操作の経験が少ない作業者

専門的な法律用語は使わず、現場の言葉で書いてください。（制約型）
```

このように複数の型を組み合わせると、**「誰に・何を・どんな形で・どんな制限で」が明確**になり、そのまま使える回答が返ってきます。

## 製造業・事務職の業務別プロンプト例10選

### 1. 週次報告書の下書き
```
今週の業務報告書を作成してください。
対象: 上司への提出用
形式: 実施業務・課題・来週の予定の3項目・各200字以内
内容:
- 受注管理データの更新（50件）
- 月次集計表の完成・提出
- 取引先3社への見積書送付
```

### 2. メールの丁寧語チェック
```
以下のメールを、取引先への送付にふさわしいビジネス文書に書き直してください。
過度に堅くなりすぎず、自然な丁寧語でお願いします。
[メール本文を貼る]
```

### 3. 会議アジェンダの作成
```
来週の品質会議のアジェンダを作成してください。
会議の目的: 先月の不良品発生原因の共有と再発防止策の決定
参加者: 製造部・品質管理部・設備管理部の課長級（計5名）
所要時間: 60分
形式: 時間配分付きの箇条書き
```

### 4. マニュアルの平易化
```
以下の手順書を、入社1年目の新人でも理解できる言葉に書き直してください。
専門用語には（）で簡単な説明を加えてください。
[手順書テキストを貼る]
```

### 5. クレーム対応メールの下書き
```
取引先からクレームを受けた際のお詫びメールを作成してください。
状況: 納品した部品に寸法誤差があり、相手方の生産ラインが一時停止した
トーン: 誠意ある謝罪・原因調査中であることを伝える・再発防止策は後日連絡
文字数: 400字以内
```

### 6. 5S活動報告のまとめ
```
以下のメモをもとに、5S活動の月次報告書を作成してください。
形式: 実施内容・改善前後の比較・次月の目標の3項目
読み手: 工場長
[活動メモを貼る]
```

### 7. 採用面接の質問リスト
```
製造業の生産管理職の中途採用面接で使う質問リストを作成してください。
確認したいポイント: 即戦力性・コミュニケーション能力・問題解決力
質問数: 10問
各質問に「なぜこれを聞くか」の意図も添えてください
```

### 8. 社内FAQ作成
```
以下の問い合わせに対するFAQを作成してください。
想定読者: 社内の非IT部門の一般社員
形式: 質問（Q）→ 答え（A）の形式で5件
[よくある問い合わせ内容を箇条書きで貼る]
```

### 9. プレゼン構成の提案
```
以下のテーマで15分のプレゼン構成を考えてください。
テーマ: 当社での生成AI活用の提案（経営層向け）
構成: 課題→解決策→導入効果→コスト→まとめ
スライド枚数: 10枚以内
各スライドの見出しと要点を箇条書きで示してください
```

### 10. 議事録からToDoリストを抽出
```
以下の議事録から、担当者・タスク内容・期限を抽出して
表形式のToDoリストを作成してください。
担当者が不明なものは「未定」と記載してください。
[議事録テキストを貼る]
```

## プロンプトをうまく書くための3つの心がけ

### 心がけ①: 最初から完璧を求めない

最初のプロンプトで完璧な答えは返ってきません。「もう少し短く」「もっと具体的に」「箇条書きに変えて」と**対話しながら改善**していくのが正しい使い方です。

### 心がけ②: うまくいったプロンプトを保存する

「この指示で良い結果が出た」と感じたプロンプトはメモしておきましょう。WordやExcelのテンプレートファイルに貯めておくと、次回からコピペで使い回せます。

### 心がけ③: 確認は必ず自分でする

AIの出力は**必ず自分の目で確認**してください。特に数値・固有名詞・日付・社内ルールに関わる内容は、AIが正確に把握できていないことがあります。「下書き」として使い、最終確認は人間が行うのが鉄則です。

## まとめ——5つの型を組み合わせれば、AIは最高の仕事相手になる

| 型 | キーフレーズ | 使い場面 |
|----|------------|---------|
| ①役割型 | 「あなたは〇〇の専門家です」 | 専門知識が必要なとき |
| ②状況型 | 「〜という状況です。〜して」 | 背景を伝えたいとき |
| ③出力型 | 「〜形式で・〜字以内で」 | 形式を決めたいとき |
| ④制約型 | 「〜は使わず・〜だけに絞って」 | 余計なものを省きたいとき |
| ⑤例示型 | 「以下の例のように」 | 社内フォーマットに合わせたいとき |

まずは**型①「役割型」から試してみてください**。「あなたは〇〇の専門家です」の一文を加えるだけで、AIの回答が変わることを実感できます。
