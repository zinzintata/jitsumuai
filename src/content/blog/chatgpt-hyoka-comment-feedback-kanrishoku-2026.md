「評価コメントを書いてください」——この依頼が来るたびに、何時間も悩んでいませんか。

部下の仕事ぶりは頭に入っている。でも、それを適切な文章にするのが難しい。特に「改善を促したいけれど傷つけたくない」「良い評価なのに陳腐な言葉しか出てこない」というとき、言葉が止まってしまう——製造業の現場リーダーの多くが抱えている悩みです。

この記事では、ChatGPTを使って評価コメントの「文章化」を支援する方法を、コピーして使えるプロンプトとともに解説します。AIを「文章化の補助ツール」として使い、あなた自身の観察と判断を適切な言葉にする方法です。

---

## 1. 評価コメントが書けない3つの理由

### 理由① 「何を書けばいいか」の型を知らない

評価コメントには、実は共通の型があります。しかし多くの現場リーダーはその型を教わることなく、毎回白紙から書き始めています。型を知るだけで、書くスピードが格段に上がります。

### 理由② 「良いことも悪いことも言わないといけない」プレッシャー

良い評価だけを書けば「甘い評価」と思われるかもしれない。改善点だけを書けば「ダメ出しリーダー」と思われるかもしれない。このバランスへの不安が、手を止めます。

### 理由③ 「言葉にすると感情的に見える」という懸念

現場で感じた「もっと積極性を出してほしい」という気持ちを、客観的でフェアな言葉に変換することが難しい。主観が強すぎると評価の公平性が疑われます。

AIは、これらの「文章化の壁」を取り除く補助ツールとして機能します。

---

## 2. AIが評価コメント作成を支援できる理由

### 観察はあなた、言葉はAIが補助する

評価で最も重要なのは、**「部下が何をしたか」「それがどんな価値を生んだか」をあなたが観察・判断すること**です。これはAIにはできません。

AIが支援できるのは、あなたの観察をもとに「読み手に伝わる文章」に変換する作業だけです。

この分業が明確になると、評価コメント作成はずっと楽になります。

### SBIフィードバックの型をAIが自動適用する

評価コメントで最も使われる型が**SBIフィードバック**です。

- **S（Situation）**：どんな状況・場面で
- **B（Behavior）**：どんな行動をしたか
- **I（Impact）**：それがどんな影響・結果をもたらしたか

この型に沿って書くと、具体的で客観的なコメントになります。ChatGPTにこの型を使うよう指示することで、あなたのメモを自動的にSBI形式の文章に変換できます。

## 3. ChatGPTで使える評価コメント生成プロンプト集

### プロンプト① ポジティブ評価（良い点を伝える）

```
あなたは製造業の現場リーダーの評価コメント作成を支援するアシスタントです。
以下の観察メモをもとに、人事評価向けの「ポジティブな評価コメント」を作成してください。

【条件】
・SBIフィードバックの形式（状況→行動→影響）で書く
・200〜300字程度にまとめる
・具体的で、誰が読んでも同じ場面を思い浮かべられる文章にする
・感情的・主観的な表現を避け、事実と影響に基づいて書く
・評価を受ける本人が読んでも納得できる内容にする

【評価対象者】
名前（仮）：[名前]
役職・年次：[役職・入社年次]
評価期間：[期間]

【観察メモ（箇条書きでOK）】
・[良かった行動・場面を3〜5点書く]

上記をもとに評価コメントを作成してください。
```

**使用例：**

入力メモ：
```
名前：田中さん（仮）
役職・年次：製造ライン作業者・入社4年目
評価期間：2025年下半期

・ラインの段取り替えで手順書にない問題が発生したとき、先輩に確認せず自己判断せずにすぐ報告してくれた
・新人の山田さんの教育を自発的に手伝い、山田さんの不良品率が前月比40%改善した
・月次の設備点検で過去に誰も気づかなかった微細な異常を発見し、大きなトラブルを未然に防いだ
```

出力例（AI生成）：
```
【田中さん 評価コメント（下半期）】

段取り替え中に手順書外の問題が発生した際、自己判断せず即座に上位者へ報告・確認を徹底しました。
この行動により問題の早期共有が実現し、ライン停止を防ぐことができました。

また、新人の山田さんへの教育支援を自発的に行い、不良品率を前月比40%改善する成果に貢献しました。
自身の業務と並行して後輩育成に取り組む姿勢は、チーム全体の底上げにつながっています。

月次点検では過去に未検出だった微細な設備異常を発見・報告し、大きなトラブルを未然に防ぎました。
細部への注意力と報告・連絡・相談の実践力を高く評価します。
```

### プロンプト② 改善要望（課題を前向きに伝える）

改善を求めるコメントは最も書きにくい部分です。「責めているように聞こえないか」という不安からどうしても抽象的になりがちですが、具体性がないと本人が何を改善すればいいかわかりません。

```
あなたは製造業の現場リーダーの評価コメント作成を支援するアシスタントです。
以下の観察メモをもとに、人事評価向けの「改善を促すコメント」を作成してください。

【条件】
・SBIフィードバックの形式（状況→行動→影響）で書く
・200〜250字程度にまとめる
・非難・責めるトーンを避け、「次の期間でこう成長してほしい」という期待のトーンで書く
・具体的な改善行動を1〜2つ明示する
・評価を受ける本人が「何をすれば良いか」が明確にわかる内容にする

【評価対象者】
名前（仮）：[名前]
役職・年次：[役職・入社年次]
評価期間：[期間]

【観察メモ（課題・改善点を箇条書きで3点以内）】
・[改善してほしい行動・場面]

上記をもとに改善コメントを作成してください。
```

### プロンプト③ 総合評価（ポジティブ＋改善を1つにまとめる）

```
あなたは製造業の現場リーダーの評価コメント作成を支援するアシスタントです。
以下の良い点と改善点のメモをもとに、総合評価コメントを作成してください。

【条件】
・前半で良い点（2〜3点）、後半で改善への期待（1〜2点）を書く
・全体で300〜400字程度
・「良い点を認めた上で、次のステージへの期待を示す」トーンにする
・評価期間の締めくくりとして、本人がモチベーションを持てる内容にする
・断定的な表現より「〜と評価しています」「〜を期待しています」を使う

【評価対象者】
名前（仮）：[名前]
役職・年次：[役職・入社年次]
評価期間：[期間]

【良い点（3〜5点）】
・[箇条書き]

【改善・成長への期待（1〜2点）】
・[箇条書き]

上記をもとに総合評価コメントを作成してください。
```

### プロンプト④ フィードバック面談用トーク例文

評価コメントの文章だけでなく、**面談での口頭フィードバック**に使える例文も生成できます。

```
以下の評価コメントをもとに、15〜20分の1on1フィードバック面談で使える
「冒頭の話し出し例文」を3パターン作成してください。

【条件】
・どのパターンも最初に良い点への承認から始める
・話し言葉（ですます調のカジュアルな表現）で書く
・「読み上げる」のではなく「会話のきっかけ」になる長さ（4〜6文程度）
・評価コメントの内容と矛盾しない

【評価コメント（先ほど生成したもの）】
[ここに先ほど作成した評価コメントを貼る]
```

## 4. 注意点：AIが書いたコメントをそのまま提出してはいけない

### AIはあなたの観察を「代わりに表現する」だけ

AIが生成した評価コメントは、あなたが入力した情報をもとにしています。つまり、**入力する観察メモの質が、コメントの質を決めます**。

「なんとなく仕事は普通」「とくに目立ったことはない」というメモでは、具体性のない抽象的なコメントしか生成されません。評価の核心は、あなたの観察と判断にあります。

### 最終チェックは必ず自分で行う

AIが生成したコメントを提出する前に、以下を必ず確認してください。

- [ ] 実際に観察した事実と一致しているか
- [ ] 特定の個人情報・プライバシーに触れていないか
- [ ] 評価対象者の名前・数字が正確か
- [ ] 会社の評価基準・フォーマットに沿っているか
- [ ] 本人が読んで「理不尽だ」と感じる表現がないか

AIの出力を「たたき台」として、自分の言葉に修正することを習慣にしてください。

### 個人情報の取り扱い

ChatGPTなどの生成AIに入力する情報には、部下の実名・具体的な業績数字が含まれる場合があります。会社のセキュリティポリシーに従い、実名は仮名で入力するか、社外秘情報を含まない形で使用してください。

## 5. 評価コメントの質を上げる「観察メモ」の取り方

AIの出力はメモの質に依存します。日頃からこの視点でメモを取ると、評価時に困りません。

### 月1回「評価メモ」を取るだけでいい

| メモの内容 | 具体例 |
|----------|------|
| 良い行動・場面（日付付き） | 「3/15 品質不良の早期発見・即報告」 |
| 改善が必要な場面（日付付き） | 「3/28 指示を繰り返さないと動かなかった」 |
| 数字で表せる成果 | 「4月の不良品率 0.3%→0.1%に改善」 |
| 周囲への影響 | 「チームの士気向上に貢献・後輩が相談しやすい雰囲気を作った」 |

この4項目を月1回メモしておくだけで、評価コメント作成時に「何も思い出せない」という状態がなくなります。

---

## 6. まとめ：AIは「書く補助」、判断はあなた

### この記事で伝えたかったこと

- 評価コメントが書けない原因は「型を知らない・文章化が苦手」なことが多い
- ChatGPTは「観察メモ → SBI形式のコメント」への変換を数秒で行える
- ただし、入力する観察の質と最終チェックは必ず人間（あなた）が担う
- 月1回の「評価メモ習慣」があると、評価期間末の作業が大幅に楽になる

AIを使っても、部下をよく見て・正当に評価する責任はリーダーにあります。AIはその表現を手伝うだけです。

### 今週できる最初の一歩

1. ChatGPTの無料アカウントを開く（スマホで5分）
2. 今期の部下の観察メモを箇条書きで3点書き出す
3. 「プロンプト①（ポジティブ評価）」を使って評価コメントを試作する
4. 出力を読んで「自分の言いたいことが伝わるか」を確認する
5. 修正・加筆して完成させる
